# Vasmi 工具使用指南

Vasmi 是芯片的硬件信息监测与管理工具，可用于采集底层硬件芯片设备类型、设备运行频率与温度等物理信息，查询上层应用软件运行进程以及进程对硬件资源的内存占用、编解码开销等信息，同时可以设置过流与过温保护信息。

## 功能特性

- 查询初始化设备状态信息
- 查询固件版本号、驱动版本号、应用层软件版本号
- 查询当前主机设备列表、设备型号和主机软件设备节点
- 查询设备算力资源利用情况
- 查询硬件内存空间、虚拟内存空间利用率
- 查询当前设备运行进程信息、进程内存情况
- 查询设备当前运行功率、峰值运行功率、当前功率对应的电压与电流情况
- 查询设备温度
- 查询设备运行时钟频率
- 查询 PCIe 带宽信息和 DDR 带宽信息
- 获取 Core 错误信息
- 设置设备参数（Bbox 模式、User Config、视频多核模式等）

## 运行环境

### 硬件环境

- CPU: Intel i5-4570（或更高配置），或性能相当的 AMD/ARM 架构 CPU
- Memory: 8GB 或以上

### 支持的卡类型

**SV100 系列**:
- VA1、VA1A、VA1L、VA1U、VA1V、VA10、VA10L、VE1S、VA1M、VA12、VA16、VA10S

**SGPU100 系列**:
- VG1000、DC1000、VG1000c、DC1000c

### 软件环境

- Ubuntu 18.04/20.04 (x86 和 ARM)
- CentOS 7.2+ (x86 和 ARM)
- OpenEuler 22.03 (ARM)
- NDK
- Windows 64 位

## 安装步骤

### 自动安装

一般情况下，业务软件安装完成后，工具的依赖库和工具自动安装。

### 手动安装

1. 解压安装包
```bash
tar -xzvf tools-xxx-fusion-tools.tar.gz
```

2. 安装 Tools
```bash
cd tools-xxx-fusion/tools
./Tools-xxx-sv100.bin
```

显示 "Tools install succeeded." 说明安装成功。

3. 设置环境变量
```bash
source /etc/profile
```

注意：当前设置的环境变量仅在当前会话有效。以下情况需重新执行：
- 系统重启
- 用户关闭终端窗口
- 用户重新登录

4. 验证版本
```bash
vasmi --version
```

## 卸载

切换至安装目录，执行卸载脚本：
```bash
cd /opt/va/vaststream
./uninstall.tools.sh
```

## 命令参考

### 参数使用说明

| 参数 | 说明 |
|------|------|
| -d <device_id> | 指定设备 ID |
| -i <die_id> | 指定 Die ID |
| -p | 指定进程 ID |
| --timestamp | 显示时间戳 |
| --help | 显示帮助信息 |
| --version | 显示版本信息 |

### 设置类命令

#### 启动两个 RDMA 网卡压测

```bash
vasmi rdma-test -d 0,1
```

#### 设置加速卡 Bbox 模式

```bash
vasmi set-bbox <mode>
```

模式说明：
- 0: 正常模式
- 1: Bbox 模式

#### 设置 User Config 信息

```bash
vasmi set-user-config <key> <value>
```

#### 设置视频多核模式

```bash
vasmi set-video-mode <mode>
```

#### 设置视频编解码权重

```bash
vasmi set-video-weight <encode_weight> <decode_weight>
```

#### 设置 VA1U 加速卡固定频率

```bash
vasmi set-freq -d <device_id> --fixed <frequency>
```

#### 通过 PCIe 复位并重启设备

```bash
vasmi reset -d <device_id>
```

## 常见错误提示

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| E001 | 设备未找到 | 检查设备 ID 是否正确 |
| E002 | 权限不足 | 使用 sudo 或以 root 用户运行 |
| E003 | 参数错误 | 检查命令参数格式 |
| E004 | 设备忙 | 等待当前任务完成后再操作 |
| E005 | 不支持的命令 | 检查工具版本是否支持该命令 |
| E006 | 超时 | 检查设备连接状态 |

## 注意事项

1. 设置类命令可能需要 root 权限
2. 修改设备配置前建议先备份当前配置
3. 复位设备会导致当前运行的任务中断
4. 设置固定频率可能影响设备散热和稳定性
5. 监控命令会持续输出，使用 Ctrl+C 停止
6. RDMA 相关功能需要特定硬件支持

## 故障排查

| 问题 | 解决方案 |
|------|----------|
| 无法识别设备 | 检查驱动是否正确安装，使用 `vasmi list` 确认 |
| 查询数据为 0 | 确认设备是否正常工作 |
| 设置命令失败 | 检查是否有足够的权限 |
| 监控输出异常 | 尝试更新到最新版本 |
| 温度过高警告 | 检查散热系统，降低负载 |
| PCIe 带宽低 | 检查 PCIe 插槽和线缆连接 |
